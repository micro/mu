# x402 payments

Mu accepts x402 micropayments on its MCP endpoint: an agent can call a metered
tool with no account, receive an HTTP `402` advertising the price and pay-to
address, pay in USDC on Base, and retry. This note records how it works today,
its overlap with go-micro's own x402 support, and the plan to converge on the
framework (dogfooding, per the thesis).

## How it works today

Two ways to pay a metered tool:

- **Logged in → credits.** A session/PAT draws from the account's credit
  balance (`wallet.CheckQuota` / `ConsumeQuota`).
- **Anonymous → x402.** No session: the caller gets a `402` challenge and pays
  per call.

The gate lives at the HTTP layer in `main.go` (not inside the go-micro MCP
resolver) because `/mcp` is a public route and the policy is *credits OR x402* —
a decision that needs the auth + wallet packages, which the resolver callback
does not have. Flow for `POST /mcp`:

1. Peek the JSON-RPC body; `api.MCPWalletOp` returns the metered tool's wallet
   op (or "" for free tools / non-tool-calls).
2. If a payment header (`X-PAYMENT` or `PAYMENT-SIGNATURE`) is present → mark the
   request for settlement and wrap the writer so the settlement lands in
   `X-PAYMENT-RESPONSE`.
3. Else if there is no valid session → `wallet.WritePaymentRequired` emits the
   standard `402` (`{x402Version, error, accepts:[…]}`).
4. Else → fall through to the credit path.

The wire (`wallet/x402.go`) is spec-conformant "exact" scheme: atomic
`maxAmountRequired`, CAIP-2 network, `payTo`, `asset`, `maxTimeoutSeconds`, and
the EIP-712 `extra{name,version}`. Verify+settle relay the client's payload to
the facilitator as `{paymentPayload, paymentRequirements}` and parse
`isValid` / `success`.

### Facilitator + mainnet

The default facilitator is Coinbase CDP
(`api.cdp.coinbase.com/platform/v2/x402`), which can settle real USDC on Base
mainnet — the open `x402.org` facilitator is testnet-only. CDP's verify/settle
require a per-request **Ed25519 Bearer JWT** minted from `CDP_API_KEY_ID` /
`CDP_API_KEY_SECRET` (`wallet/x402_cdp.go`, stdlib crypto — no new dependency).

Config (all env):

| Var | Default | Notes |
|-----|---------|-------|
| `X402_PAY_TO` | — | receiving address; enables x402 when set |
| `X402_FACILITATOR_URL` | CDP | verify/settle endpoint |
| `X402_NETWORK` | `eip155:8453` | accepts `base` / `base-sepolia` too |
| `X402_ASSETS` | `USDC` | comma-separated symbols |
| `CDP_API_KEY_ID` / `CDP_API_KEY_SECRET` | — | required only for CDP |

`mu x402` prints this config and, when CDP creds are present, mints a JWT and
probes CDP `/supported` to certify auth — without printing the secret.
`deploy/verify-x402.sh <host>` checks the challenge half from outside.

## Overlap with go-micro's x402

go-micro (v6.3.10) already ships x402: `wrapper/x402` (Config, Requirements,
`Require`, `HTTPFacilitator`) and native gating in `gateway/mcp`
(`Payment *x402.Config`, per-tool `Amounts`, `PaymentInfo` advertised in the
tool catalog). **Mu's `wallet/x402.go` is a parallel implementation of this.**
The pieces Mu adds are exactly the gaps in the framework's version:

| Gap in go-micro `wrapper/x402` | What Mu already does |
|---|---|
| `Facilitator` is verify-only; `HTTPFacilitator` never calls `/settle` → funds authorized, never captured | verify **and** settle |
| `HTTPFacilitator` carries no auth → cannot reach CDP → no mainnet | Ed25519 CDP Bearer JWT |
| `Network` defaults to `"base"` (v1 short name), no CAIP-2 | `eip155:8453` (+ normaliser) |
| `Requirements` has no `extra` (EIP-712 domain) | `extra{name,version}` per asset |
| Only reads `X-PAYMENT` | also `PAYMENT-SIGNATURE` |

## Target end-state (converge on the framework)

1. **Upstream the gaps into go-micro** `wrapper/x402`: a settling `Facilitator`,
   a CDP facilitator (Ed25519 Bearer auth), CAIP-2 networks, `extra`/EIP-712 in
   `Requirements`, and `PAYMENT-SIGNATURE` acceptance. Mu's `wallet/x402*.go` is
   the reference for all five.
2. **Thin Mu down** to: construct an `x402.Config`, inject the CDP `Facilitator`,
   and keep only the *policy* (logged-in → credits, else → x402). The wire
   (challenge/requirements/relay/headers) becomes go-micro's, not Mu's.
3. Mu stays the dogfood that keeps the framework's x402 honest.

Until (1) lands, Mu's local implementation is the source of truth — it is
shipped, conformant, and settles on mainnet.

## Pricing surfaces

One cost source (`wallet.GetOperationCost`, where 1 credit ≈ 1¢) rendered two
ways: **credits** for humans on `/pricing`, and **USDC per-call** for agents
inline on `/mcp` (via `wallet.X402PriceFor`, shown only when x402 is enabled).
There is deliberately no separate developer pricing page — the per-tool prices
on `/mcp` are the price list.
